# Task 40: Mobile Hero Fallback Implementation

## Completed: 2026-02-18

## Changes Made

### File Modified: `src/components/sections/Hero.astro`

1. **Added mobile/desktop background switching using CSS media queries:**
   - Mobile (`< md`): Shows static image
   - Desktop (`>= md`): Shows YouTube video (or image if configured)

2. **Added fallback chain for mobile image:**
   - Priority 1: `mobile_image_url` from data
   - Priority 2: `image_url` from data
   - Priority 3: YouTube poster image (`img.youtube.com/vi/{videoId}/maxresdefault.jpg`)

3. **Updated Props interface:**
   - Added `background_type?: 'youtube' | 'image'`
   - Added `youtube_id?: string`
   - Added `video_id?: string` (legacy support)
   - Added `image_url?: string`
   - Added `mobile_image_url?: string`

## Benefits
- Better mobile performance (no video loading)
- Avoids autoplay issues on mobile devices
- Reduces data usage for mobile users
- Maintains full video experience on desktop
